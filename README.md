# Py_liepin_jobname_spider
基于Python requests爬取猎聘网招聘岗位数据

# 思考与总结
1.保存相同类型的数据，数据包有什么变化
  发送请求的接口没有发生变化，变化的只有对应数据包的请求参数
  本质上是后端数据库'WHERE'子句筛选条件发生了改变
2.批量采集和单个数据包采集有哪些区别
  相对于单个数据包来说，批量采集其中的部分参数会发生改变，所以在采集步骤上面
  需要加上发送请求参数的变量
3.特殊情况如何分析
    3.1 Python运行的结果是报错还是响应包出现了问题，需要在哪里找错误
      报错-出现红色字体
      响应包出现问题-如{"flag":0,"code":"-1400","msg":"出错了（400）！"}
      解决方法是去查看请求头或是传递的参数这两个变量是否没有齐全
    3.2 批量数据采集，对应的request.get/post()中的形参名应该如何确定
      需要看标头-响应标头Content-Type:
      content-type:application/json;charset=UTF-8
      出现json则对应形参名为json
    3.3 参数值为 `None` 的处理方案
      若待传入的请求参数值为'None'，该参数无法直接提交至接口，需要删除该参数键值对

# 思路
1.分析接口
   抓包分析猎聘搜索接口，
   发现接口地址固定，仅关键词、页码参数变化
2.请求头构建
   复制浏览器真实请求头（User-Agent、Cookie、Referer等），绕过基础反爬
3.构建参数变量
   将搜索关键词 key、翻页页码 currentPage 设置为可修改变量
4.批量请求发送
   通过循环遍历页码，使用 requests.post 发送 JSON 格式批量请求
5.数据提取
   将接口返回的 JSON 数据解析，循环提取每一条岗位名称信息。
6.保证稳定性
   加入 time.sleep(1) 延时，降低请求频率，保证程序稳定运行

# 关键点
1.翻页：页面展示第1页，对应currentPage=0
2.反爬意识：必须携带完整请求头，否则报错
3.数据遍历：使用双层循环，外层翻页，内层遍历岗位列表，获取全部数据
